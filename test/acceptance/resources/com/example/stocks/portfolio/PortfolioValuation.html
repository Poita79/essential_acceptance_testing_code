Portfolio Valuation

HTTP Adapter to Java Message Test

Implementation notes

The Portfolio's adapter is implemented in terms of HTTP; it's a RESTful server which collaborates with this business interface directly. It turns the HTTP API into a Java API; we're testing that a HTTP message turns into a Java message.

This test is implemented with the assumption that RESTful infrastructure is tested elsewhere. That is to say, rather than startup the server and send a real HTTP request from the test, the component it delegates to (in our case the PortfolioResource.java class) is tested in isolation.

An alternative option is to startup the server and send a real HTTP message in but this could add overhead when multiple scenarios need to be tested. In both cases, a test double can be used to verify the (HTTP) message is translated in a Java message.

The calculation logic is not tested here.

When

A HTTP request for a valuation is received

Then

The total portfolio valuation will be calculated a returned to the requester
